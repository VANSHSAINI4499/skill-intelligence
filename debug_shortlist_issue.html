🔍 Company Shortlist Debug Tool

❌ Current Issue

Error: Company 'GOOGLE' not found

Root Cause: Frontend sending company name instead of UUID

🚀 Quick Fix Steps

Step 1: Check Browser Network Tab

1. Open Chrome DevTools (F12) → Network tab

2. Go to Admin → Shortlists page

3. Look for this API call:

GET /api/admin/company-requirements

4. Click on it → Response tab

Expected response format:

[{
    "companyId": "abc123-4567-uuid-here",    ← Should be UUID
    "companyName": "GOOGLE",                 ← Display name
    "minCGPA": 7.0,
    "minLeetCodeHard": 50,
    ...
}]

Step 2: Try Generate Shortlist

1. Select "GOOGLE" from dropdown

2. Enter batch year (e.g., "2023")

3. Click "Generate Shortlist"

4. In Network tab, look for:

POST /api/admin/shortlists

5. Click on it → Payload/Request tab

Check the payload:

{
    "companyId": "???",    ← What is this?
    "batch": "2023",
    "limit": 20
}

Step 3: Identify The Problem

✅ If payload shows UUID:

{"companyId": "abc123-4567-uuid"}

→ Backend database issue. Company doesn't exist with that UUID.

❌ If payload shows company name:

{"companyId": "GOOGLE"}

→ Frontend bug. Dropdown not sending correct UUID.

🛠️ Likely Fixes

Fix #1: Company Wasn't Created Properly

If the dropdown is empty or only shows "GOOGLE" but no UUID in the network call:

  1. Go to Admin panel
  2. Create a new company requirement properly
  3. Make sure it generates a UUID
  4. Refresh shortlist page

Fix #2: Frontend State Bug

If the network call shows company name instead of UUID:

// Quick fix: Check for typos in frontend code
// File: frontend/app/admin/shortlists/page.tsx
// Line 160: value={c.id} should NOT be value={c.companyName}

Fix #3: Manual Input Bug

Make sure you're selecting from the dropdown and NOT typing "GOOGLE" manually in the company field.
